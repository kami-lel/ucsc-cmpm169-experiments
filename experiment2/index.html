Experiment 2 - Hexagon Drawing

Move mouse around to change shapes of the hexagons.
Click the screen to change colors.

Description

Hexagon Drawing creates interactive vector art and animation by using mouse position to control line attributes and integrating hexagonal patterns.

Technical

There are some interesting points from a technical perspective. Despite being simple vector art, when the player decreases the size of the hexagon, more hexagons are required to fill the entire screen. During testing, it became apparent that if too many hexagons are drawn, it consumes a significant amount of computational power, resulting in a less smooth experience. Therefore, I needed to write code to optimize the location calculation process. Instead of using sine and cosine functions every time I want to draw a stroke, I calculate an x offset and y offset for each frame. I structured the mathematics so that all x-y locations can be calculated using simple addition without invoking trigonometry. This drastically improves performance. When attempting to introduce randomness into the pattern of the hexagons, I cannot rely on a random number generator to directly dictate each pattern. Doing so would mean that for each frame, every stroke would change color. However, to maintain visual consistency, I must implement a pseudo-random number generator.
